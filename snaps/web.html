Flaunt

A CMS to showoff

January 2011

Flaunt is a server-side web application or CMS, which uses PHP to present and MySQL to store the content. It supports themes, which use a simple API to pull in content. All content is managed inside a backend. pedroandsybil.wgallia.com is built with it.

The idea for this came when I wanted to make a portfolio website for myself, but it was never made until I had to document a project for a University project. It was never used for a personal website...

Some of the cool features include:

  • The order of the sidebar items can be ordered in the browser by dragging and dropping boxes.
  • All .css and .js files are automatically included when they are in the correct directory.
  • The main stylesheet can be edited without any FTP/Terminal nonsense in the backend.
  • Loads of others...

The source can be downloaded/viewed on github, and I also made a profile on ohloh for it, where you can see an analysis.

Fabricate

A website maker

July 2011

Why

This was an excuse to work on my Python and JavaScript, towards the end of the project I incorporated shell scripts too, which I had never really used before. For educational purposes, no libraries were used.

It was the time of year, when I felt like I needed to revamp my website. I wanted to write something that would allow me to add content to the site easily and to hide to tedium of moving files to the server, updating, etc.

What

The workflow using Fabricate is simple, when you want to add a new section, you write a text file with a header, a header would look something like this:

name: Fabricate
cats: Web
tags: Python, JavaScript, shell, JSON

After leaving an empty line after the header, you can write the content. When you are done simply call make and the website you see in front of you is made and pushed to the server.

How

The main script is written in Python, this reads all the files and makes one big JSON object and writes it to a file. Next another Python script reads the content files to see if they have been modified since the last push, it also sees if there are any new files such as images that have been added to a content folder. Then all new files are tarred and pushed to the remote server where they are unpacked and moved to the right place.

All content on the site is created dynamically with JavaScript using the JSON file, pages are collected using XMLHttpRequests.

Pages are served using mod_python, because there is no normal crawlable content, if Google visits the page, an HTML snapshot is served which has been produced using phantomJS.

ItoA

Image to ASCII

June 2011

During the summer of 2011 I wanted to learn Python and needed a small project, this was what I chose. Using a web interface you can submit an image, either on your filesystem or using a URL, and an ASCII representation is returned.

You can try it with your own images here.

The form is processed by a small PHP script, this saves the image on the server, and calls the Python script to create the ASCII file. When the script returns the PHP prints the contents of the file.

The returned text can either be formatted to be viewed in the browser or in a text document, where the line height is different.
